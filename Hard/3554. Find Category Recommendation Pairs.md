## 🚀 3554. Find Category Recommendation Pairs

### 💡 Idea

The goal is to find pairs of **product categories** that are purchased by the same customers.

We need to determine:

- All category pairs where **`category1 < category2`**.
- The number of **distinct customers** who purchased products from both categories.
- Return only those category pairs purchased by at least **3 different customers**.

---

## 💻 Solution (Approach 1: Self Join)

```sql
SELECT pi1.category AS category1, pi2.category AS category2,
COUNT(DISTINCT pp1.user_id) AS customer_count
FROM ProductPurchases pp1
INNER JOIN ProductPurchases pp2 ON pp1.user_id = pp2.user_id
INNER JOIN ProductInfo pi1 ON pi1.product_id = pp1.product_id
INNER JOIN ProductInfo pi2 ON pi2.product_id = pp2.product_id
WHERE pi1.category < pi2.category
GROUP BY category1, category2
HAVING customer_count >= 3
ORDER BY customer_count DESC, category1, category2;
```

### 🛠️ How It Works

1. Self join the `ProductPurchases` table on `user_id` to compare products purchased by the same customer.
2. Join the `ProductInfo` table twice to retrieve the category of each product.
3. Use `pi1.category < pi2.category` to generate each category pair only once.
4. Count the **distinct customers** who purchased products from both categories.
5. Keep only category pairs purchased by at least **3 customers** and sort the result.

---

## 💻 Solution (Approach 2: CTE + Self Join)

```sql
WITH user_categories AS (
    SELECT DISTINCT pp.user_id, pi.category
    FROM ProductPurchases pp
    JOIN ProductInfo pi ON pi.product_id = pp.product_id
),
category_pairs AS (
    SELECT uc1.user_id, uc1.category AS category1, uc2.category AS category2,
    FROM user_categories uc1
    JOIN user_categories uc2 ON uc1.user_id = uc2.user_id AND uc1.category < uc2.category
)
SELECT category1, category2, COUNT(DISTINCT user_id) AS customer_count
FROM category_pairs
GROUP BY category1, category2
HAVING COUNT(DISTINCT user_id) >= 3
ORDER BY customer_count DESC, category1, category2;
```

### 🛠️ How It Works

1. The `user_categories` CTE creates a unique list of **(user, category)** pairs.
2. The `category_pairs` CTE self joins this list to generate all category pairs purchased by each user.
3. Group by the category pair and count the **distinct users**.
4. Keep only category pairs purchased by at least **3 customers** and sort the result.

---

## 🔄 Approaches

### ✅ Approach 1: Self Join

This approach directly generates category pairs from the purchase records.

#### Advantages

- Short and simple query.
- Easy to understand.
- Good for small and medium-sized datasets.

#### Time Complexity

**O(n²)**

- `n` = total purchase records.

#### Space Complexity

**O(n)**

---

### ✅ Approach 2: CTE + Self Join ⭐ (Recommended)

This approach first creates a unique list of **(user, category)** pairs before generating category pairs.

#### Advantages

- Removes duplicate purchases from the same category for a user.
- Reduces the number of rows before the self join.
- Cleaner and more modular logic.
- Better suited for large datasets where users may purchase multiple products from the same category.

#### Time Complexity

**O(u²)**

- `u` = number of unique **(user, category)** pairs.

#### Space Complexity

**O(u)**

---

### ⭐ Recommended Approach

**Approach 2 (CTE + Self Join)** is recommended.

Although it uses an additional CTE, it is generally more efficient because it removes duplicate `(user, category)` combinations before generating category pairs. This reduces unnecessary comparisons during the self join and makes the query easier to read and maintain.

**Approach 1** is a good solution but it may process more rows when users purchase multiple products from the same category.

---

### ⏱️ Complexity Summary

| Approach | Time Complexity | Space Complexity |
|----------|-----------------|------------------|
| Self Join | **O(n²)** | **O(n)** |
| CTE + Self Join ⭐ | **O(u²)** | **O(u)** |

Where:

- **n** = total purchase records.
- **u** = unique `(user, category)` pairs (`u ≤ n`).
