## 🚀 3497. Analyze Subscription Conversion

### 💡 Idea

The goal is to find users who converted from a **free trial** to a **paid subscription** and calculate their average activity duration during both periods.

We use **conditional aggregation** with `CASE` inside the `AVG()` function to calculate both averages in a single query. Finally, we keep only users who have a paid subscription.

---

## 💻 Solution

```sql
SELECT user_id,
ROUND(AVG(CASE WHEN activity_type='free_trial' THEN activity_duration ELSE NULL END),2) AS trial_avg_duration,
ROUND(AVG(CASE WHEN activity_type='paid' THEN activity_duration ELSE NULL END),2) AS paid_avg_duration
FROM useractivity
GROUP BY user_id 
HAVING paid_avg_duration is not null
ORDER BY user_id
```

---

### 🛠️ How It Works

1. Group all records by `user_id`.
2. Use `CASE` inside `AVG()` to calculate the average activity duration for **free trial** and **paid** activities separately.
3. Round both averages to **2 decimal places** using `ROUND()`.
4. Use `HAVING` to keep only users who have a paid subscription.
5. Sort the result by `user_id`.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)`

**Space Complexity:** `O(n)` *(for grouping by user)*
